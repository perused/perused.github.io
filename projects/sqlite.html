SQLite to T-SQL

A program that converts an SQLite schema into a T-SQL graph as per Microsoft SQL-Graphs.

Description

When exploring the pathways for creating Dom2GE, it became necessary to see how SQL could be represented as a graph. One option was to use T-SQL graphs. However, the schemas being worked with were written in SQLite. In order to determine the efficacy of T-SQL graphs, a conversion from SQLite schemas to T-SQL graph schemas was created. Each node was a row in a table and each edge is constructed between nodes. It is possible to tweak the program to allow edges to only be between primary and foreign keys as well.

In the end, a heterogeneous graph was used to represent SQL in the Dom2GE methodology. However, pathways such as this one are still considered as important future work for improving upon the heterogeneous graph representation.

Since this branch of the project never reached the final stages, the codebase is still littered with comments and requires refactoring. Regardless, the repository can be viewed here.
